Rollback Plan

Rollback Triggers

  • Error rate exceeds 2%
  • AI service unavailable for > 5 minutes
  • Critical data loss bug discovered
  • Docker environment instability
  • Make targets consistently failing

Rollback Procedure

  1. Set ai_planning_features flag to false in all environments
  2. Notify all stakeholders via Slack #ai-development
  3. Stop and restart Docker containers with previous image
  4. Revert git commits if code changes are problematic
  5. Document issue in incident report
  6. Schedule post-mortem within 24 hours
